- Built administrative portals: `NewAthletes.tsx` (registration approvals list), `AcademyManagement.tsx` (nested accordion views), and `AdminNotifications.tsx` (expiring memberships).

# Decisions Made

- **Sport & Group clean encapsulation**: Sport and Group were implemented as full models under `departments` app instead of using flat tags.
- **Top-level router paths for sports/groups**: Re-routed `/api/sports/` and `/api/groups/` at the root URL level because DRF DefaultRouter empty prefixes conflict with sibling `/subpath/` patterns.
- **Webcam-only canvas upload**: Enforced zero-file-input rules for selfies by leveraging Canvas base64 capture directly from device-facing webcam streams.
- **Insulated Visibility**: Self-registered players stay `is_active=False` and isolated from primary list querysets until explicitly approved by administration.
- **Package sorting**: Ordered dynamically using `Case/When` inside DRF ViewSets rather than relying on frontend-side sorting.
- **Cross-dashboard locking**: Implemented bidirectional `useEffect` redirect locks preventing parents/athletes from seeing `/dashboard`, and administrative staff from seeing `/user`.

---

# Problems Encountered

1. **DRF nested subpath 404s**: Standard DefaultRouter registration of `""` pattern on `/api/departments/` was catching sibling `/sports/` and `/groups/` paths as a primary key lookup.
2. **Pytest dotenv environment warnings**: Running pytest directly did not trigger `manage.py`'s environment-variable hydration, causing a runtime secret key error in `base.py`.
3. **TypeScript context & interface compilation errors**: Modifying `auth.tsx` `login` return types and `SubscriptionPackage` parameters threw schema mismatch exceptions in `Login.tsx` and `Memberships.tsx`.

---

# Solutions

1. Moved sport and group routes to top-level paths (`/api/sports/` and `/api/groups/`) in `config/urls.py`.
2. Created a root `backend/conftest.py` that loads dotenv globally prior to pytest testing.
3. Updated `AuthContextType` and converted all legacy `duration_days` calculations inside `Memberships.tsx` to handle `duration_type` / `duration_value` dynamically.
