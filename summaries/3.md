# Problems Encountered

1. **audit_project.py regex crash**: Pattern `(r'\bclass\b\s+...` had `ValueError: bad escape \c` due to malformed raw string — the `r` prefix was inside the regex group instead of before the string
2. **audit_project.py walked hidden directories**: `.git`, `.dart_tool`, `build/` directories were being scanned, inflating results with system files
3. **audit_project.py line counting wrong**: Used `len(content.splitlines())` which misses the final line if no trailing newline — switched to `content.count('\n')`
4. **audit_project.py output format**: Single `Results for: {pattern}` line made results hard to read — added `[FILE]` and `[TOTAL]` markers per pattern for clarity
5. **analysis_options.yaml `deprecated_member_use` error**: Rule doesn't exist in Dart 3.7 — replaced with `deprecated_member_use_from_same_package`
6. **iOS build failure**: `v1/resolver.dart` error is a pre-existing CocoaPods issue, not caused by our changes
7. **flutter analyze reported ~340 issues**: After applying strict lint rules, the existing codebase shows 340+ pre-existing issues — these will be eliminated by the rewrite (user was informed)
8. **Mobile Best Practices skill had paths missing**: Skill referenced `scripts/` and `reference/` directories but only the base directory exists
9. **App store cart/wishlist screens missing**: Backend has StoreItem and Cart endpoints but no mobile screens — noted for future

---

# Solutions

1. Rewrote `audit_project.py` from scratch: proper raw strings with `r'pattern'`, exclude hidden dirs via list, correct `content.count('\n')` line counting, per-pattern `[FILE]`/`[TOTAL]` markers
2. Replaced `deprecated_member_use` with `deprecated_member_use_from_same_package` in both `analysis_options.yaml` files
3. Used comprehensive codebase analysis (task agents) to map all models, endpoints, screens, and providers before designing the rewrite plan
4. Created detailed per-screen issue matrix to ensure no screen-level issues are missed in the rewrite
