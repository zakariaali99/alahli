# Decisions Made

- **Skip API versioning**: Single-client system, all clients controlled internally
- **Skip CI/CD for now**: Hold for later implementation
- **Arabic-only for mobile**: No i18n needed now
- **Remember Me**: Backend issues 30-day refresh token when `remember_me=True`; frontend saves phone to localStorage
- **No API URL fix in mobile**: User explicitly excluded it
- **Split requirements**: Dev-only tools (pytest, factory-boy) in dev.txt; prod-only (gunicorn, psycopg2) in prod.txt
- **Token blacklist approach**: Use `BlacklistedToken.objects.get_or_create(token=token)` instead of `RefreshToken(token.jti)` which fails
- **Single-flight refresh lock**: Prevent concurrent 401 race conditions in frontend

# Problems Encountered

1. **URL system check failure** after renaming AttendanceLogViewSet queryset rename — fixed by adding `basename="attendance"` to router registration
2. **SubscriptionViewSet same basename conflict** — added `basename="subscription"` to router
3. **NotificationViewSet same basename conflict** — added `basename="notification"` to router
4. **`OutstandingToken.blacklist()` AttributeError**: `RefreshToken(token.jti)` fails because `jti` is a UUID string, not a full JWT — silently swallowed by bare `except Exception: pass`
5. **`logout_view` and `change_password_view` both had the broken token blacklist pattern** — same bug in 3 places
6. **Test failure `test_change_password_success`**: `AttributeError: 'OutstandingToken' object has no attribute 'blacklist'`
7. **Import sorting issue in accounts/views.py** — ruff auto-fixed
8. **`SIM105` try/except in progress/views.py** — replaced with `contextlib.suppress(ValueError)`
9. **Mobile token refresh never saved new refresh token** — with `ROTATE_REFRESH_TOKENS=True`, second refresh always failed
10. **API default URL `10.22.95.35`** — private IP breaks production builds (excluded from fix per user request)
11. **Frontend concurrent 401 race condition** — concurrent refresh calls caused second to send blacklisted old token
12. **Analytics revenue underreported** — only counted ACTIVE subscriptions, ignored all renewal payments
13. **Workouts URL shadowing** — manual `as_view()` at `""` path shadowed router root
14. **Mobile homepage broken card** — error callback silently discarded error and showed misleading "no subscription" card instead of actual error
15. **`WhatsAppMessage` table missing on test**: `test_delete_athlete` fails because `notifications_whatsappmessage` table doesn't exist (pre-existing issue from other tab work)

# Solutions

1. Added `basename` parameters to all router registrations without class-level queryset
2. Replaced `RefreshToken(token.jti)` with `BlacklistedToken.objects.get_or_create(token=token)` in 3 places
3. Changed `except Exception: pass` to proper token blacklisting
4. Updated `LoginSerializer` to accept `remember_me` bool; when `True`, uses `refresh.set_exp(lifetime=timedelta(days=30))`
5. Fixed mobile refresh: `_refreshToken = asString(refreshData['refresh'])` and added `_onTokensRefreshed` callback
6. Added `refreshPromise` singleton for concurrent 401 handling in `api.ts`
7. Fixed analytics: `total_revenue = subscription_revenue + renewal_revenue`; `revenue_by_month` union-merges subscription and renewal data by month
8. Replaced manual `as_view()` paths with router registration: `router.register(r"sessions", WorkoutSessionViewSet, basename="session")`
9. Rebuilt mobile error card: `_buildErrorCard` detects `DioExceptionType` and shows appropriate error with retry; `_buildEmptyCard` is polished gradient card
10. Created `conftest.py` at `apps/` level with shared fixtures

# Important Context

- `DJANGO_SECRET_KEY` is **required** at runtime — will crash without it
- Default `ALLOWED_HOSTS` changed from `["*"]` to `["localhost","127.0.0.1"]`
- CORS is locked down: `CORS_ALLOW_ALL_ORIGINS = False`, `CORS_ALLOWED_ORIGINS` must be set in production
- JWT access token lifetime: 30 min (reduced from 1 hour); refresh: 14 days (reduced from 30 days, or 30 days if `remember_me=True`)
- Login rate-limited to 10 attempts/minute via `LoginRateThrottle`
- 6 ViewSets now require `IsReceptionOrAbove` for write operations
- `AnnouncementViewSet` is ReadOnly — announcements created via Django admin only
- Mobile `API_BASE_URL` default `http://10.22.95.35:8000/api` must be overridden at build time with `--dart-define`
- All 13 Django apps now have proper `AppConfig` with Arabic `verbose_name`
- Backend has 0 ruff errors, 0 Django check issues, 30/30 pytest tests passing
