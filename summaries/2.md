# Problems Encountered

1. **Workout sessions 405 error**: POST to `/api/sessions/` returned `405 Method Not Allowed` — router registered `sessions` prefix but include at `api/sessions/` created double prefix `/api/sessions/sessions/`
2. **Trainer reviews 405 error**: POST to `/api/trainers/reviews/` returned 405 — `TrainerViewSet` with empty prefix caught `reviews` as a pk param via `^{(?P<pk>[^/.]+)}$` regex
3. **FAQ test assertion error**: `len(response.data) == 1` failed with value 4 — forgot DRF pagination wraps results in `{"count": N, "results": [...]}`
4. **Packages same pagination issue**: Same pagination pattern missed in packages test
5. **Analytics revenue endpoint 404**: Used URL `/api/analytics/revenue-by-month/` but actual URL is `/api/analytics/revenue/`
6. **Athlete creation test 400**: Missing required `birth_date` field in POST data
7. **Attendance log test 404**: Wrong URL `/api/attendance/` vs `/attendance/` (router added at root level)
8. **Attendance log test 400**: Sent `date` field that doesn't exist on AttendanceLog model (uses `checked_in_at` DateTimeField)
9. **Trainer review test 403 instead of 201**: Reception user had no associated Athlete profile, so `perform_create` raised `PermissionDenied("Only athletes can create reviews.")`
10. **Review test 405**: POST returned 405 due to trainer URL conflict (problem 2)
11. **Summary skill not discoverable**: `SUMMARY_SKILL.md` was a flat file in `~/.config/opencode/skill/` instead of a directory containing `SKILL.md`
12. **Sentry import error**: `manage.py check` and pytest failed after adding `from config.sentry import init_sentry` — `sentry-sdk` not yet installed

---

# Solutions

1. Changed workouts include from `api/sessions/` to `api/` in `config/urls.py` so router prefixes work correctly
2. Split trainer URLs into 2 routers with `reviews/` path before `""` trainer path
3. Changed assertions to use `response.data["count"]` and `response.data["results"][0]` for paginated responses
4. Fixed analytics test URL from `/api/analytics/revenue-by-month/` to `/api/analytics/revenue/`
5. Added `birth_date: "2000-01-15"` to athlete POST data
6. Changed attendance URLs from `/api/attendance/` to `/attendance/`
7. Removed `date` field from attendance POST data
8. Created `athlete_user` fixture that links reception user to an Athlete profile for trainer review tests
9. Installed `sentry-sdk` via pip to resolve import error
10. Moved `SUMMARY_SKILL.md` into `~/.config/opencode/skill/summary-skill/SKILL.md` directory structure
